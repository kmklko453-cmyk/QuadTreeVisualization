1) Renderer.h
==================================================

네임스페이스: QuadTreeV
- 네임스페이스 내 콘솔 렌더링 관련 클래스를 정의.

전방 선언
- class ScreenBuffer;
- Renderer가 ScreenBuffer 포인터 배열을 멤버로 가지므로 헤더 간 종속성 최소화.

class Renderer
- 목적:
  - 텍스트 기반 렌더링을 위해 제출된 렌더 명령을 프레임 단위로 버퍼에 구성하고,
  - 콘솔 화면에 출력하는 매니저 클래스.
- 디자인:
  - 싱글톤 패턴(static instance, Get())으로 전역 접근.
  - 더블 버퍼링(ScreenBuffer 2개) 사용으로 깜빡임 최소화.

중첩 타입: Frame
- 역할: 한 프레임의 픽셀(문자) 데이터 + 정렬 우선순위 배열을 보관.
- 멤버:
  - CHAR_INFO* charInfoArray: 각 좌표의 문자/속성(색상) 저장.
  - int* sortingOrderArray: 각 좌표의 렌더 우선순위 저장.
- 멤버 함수:
  - Frame(int bufferCount): 배열 할당.
  - ~Frame(): 배열 해제.
  - Clear(const Vector2& screenSize): 프레임 내용을 빈칸으로 초기화.

중첩 타입: RenderCommand
- 역할: 그릴 대상 하나를 표현하는 데이터 묶음.
- 멤버:
  - const char* text: 출력할 문자열.
  - Vector2 position: 출력 위치.
  - Color color: 글자 색.
  - int sortingOrder: 렌더 순서/깊이 값.

Renderer 공개 API
- Renderer(const Vector2& screenSize)
- ~Renderer()
- void Draw()
  - 제출된 명령을 프레임에 반영 후 현재 버퍼로 출력, 큐 비움.
- void Submit(...)
  - 문자열, 위치, 색상, 정렬값을 렌더 큐에 적재.
- static Renderer& Get()
  - 싱글톤 접근자. instance가 없으면 경고 후 중단.

Renderer private API
- void Clear()
  - 프레임/현재 버퍼를 비운 뒤 다음 렌더 준비.
- void Present()
  - 화면에서 활성 버퍼를 교체(더블 버퍼 스왑).
- ScreenBuffer* GetCurrentBuffer()
  - 현재 그릴 버퍼 객체 반환.

멤버 변수
- Vector2 screenSize: 화면 크기.
- Frame* frame: CPU측 프레임 데이터(문자/우선순위 배열).
- ScreenBuffer* screenBuffers[2]: 콘솔 핸들 기반 출력 버퍼 2개.
- int currentBufferIndex: 현재 활성 인덱스.
- std::vector<RenderCommand> renderQueue: 이번 프레임에 그릴 명령 목록.
- static Renderer* instance: 싱글톤 포인터.

--------------------------------------------------
2) Renderer.cpp
--------------------------------------------------

Frame::Frame(int bufferCount)
- charInfoArray, sortingOrderArray를 bufferCount 크기만큼 new로 할당.
- 초기화 memset으로 0 세팅.

Frame::~Frame()
- SafeDeleteArray로 charInfoArray, sortingOrderArray 해제.

Frame::Clear(const Vector2& screenSize)
- 2차원 좌표계를 1차원 인덱스로 순회:
  - CHAR_INFO::AsciiChar = ' '
  - Attributes = 0
  - sortingOrderArray = -1
- 한 프레임 전체를 비우는 함수.

Renderer::instance 정적 멤버 초기화
- Renderer* Renderer::instance = nullptr;

Renderer::Renderer(const Vector2& screenSize)
- 생성자에서 인스턴스를 자기 자신으로 설정(instance = this).
- screenSize 기반으로 Frame 생성 및 Clear.
- ScreenBuffer 2개 생성 + Clear.
- Present() 호출로 초기 활성 버퍼 설정.

Renderer::~Renderer()
- frame SafeDelete.
- screenBuffers 배열 SafeDelete.

Renderer::Draw()
- Clear() 호출.
- renderQueue 순회 처리:
  1) text가 null이면 skip.
  2) y가 화면 밖이면 skip.
  3) 텍스트 길이 0이면 skip.
  4) x 방향 가시 범위 계산(visibleStart/visibleEnd)으로 clipping.
  5) 각 visible x에 대해:
     - sourceIndex 계산(command 문자열의 오프셋)
     - frame 인덱스 계산
     - frame->sortingOrderArray[index] > command.sortingOrder 이면 기존이 더 높은 우선순위므로 덮어쓰기 skip
     - 아니면 charInfoArray에 문자/색상 기록, sortingOrder 갱신
- 완성된 frame을 현재 버퍼 Draw().
- Present()로 화면 교체.
- renderQueue clear().

Renderer::Get()
- instance가 없으면 MessageBox + __debugbreak().
- 있으면 싱글톤 참조 반환.

Renderer::Clear()
- frame->Clear(screenSize)로 CPU 렌더링 버퍼 초기화.
- GetCurrentBuffer()->Clear()로 콘솔 버퍼 초기화.

Renderer::Submit(...)
- RenderCommand command 객체 구성 후 renderQueue에 push_back.

Renderer::Present()
- SetConsoleActiveScreenBuffer()로 현재 버퍼를 실제 출력으로 설정.
- currentBufferIndex = 1 - currentBufferIndex로 토글.

Renderer::GetCurrentBuffer()
- screenBuffers[currentBufferIndex] 반환.

참고:
- Frame::sortingOrderArray는 "낮은 값이 뒤에 덮어씀" 형태의 동작.
- 텍스트 길이 기반으로 x축 clipping을 수행해 범위 초과 write 방지.

3) ScreenBuffer.h
--------------------------------------------------

class ScreenBuffer
- Windows 콘솔 핸들(HANDLE) 기반 버퍼 래퍼.
- 생성/파괴와 화면 clear/draw 책임을 가짐.

메서드
- ScreenBuffer(const Vector2& screenSize)
- ~ScreenBuffer()
- void Clear()
- void Draw(CHAR_INFO* charInfo)
- HANDLE GetBuffer() const

멤버
- HANDLE buffer: 콘솔 출력 버퍼 핸들.
- Vector2 screenSize: 버퍼 크기.

--------------------------------------------------
4) ScreenBuffer.cpp
--------------------------------------------------

ScreenBuffer::ScreenBuffer(const Vector2& screenSize)
- CreateConsoleScreenBuffer()로 새 콘솔 텍스트 버퍼 생성.
- 실패 시 MessageBoxA 후 __debugbreak().
- SMALL_RECT로 콘솔 창 영역 지정 후 SetConsoleWindowInfo.
- SetConsoleScreenBufferSize로 버퍼 크기 설정.
- 커서 정보 조회 후 bVisible = false로 커서 숨김.

ScreenBuffer::~ScreenBuffer()
- buffer가 존재하면 CloseHandle().

ScreenBuffer::Clear()
- FillConsoleOutputCharacterA()로 버퍼 전체를 공백으로 채움.
- writtenCount를 받아 몇 글자를 썼는지 추적(실무 디버그용 변수로 활용 가능).

ScreenBuffer::Draw(CHAR_INFO* charInfo)
- SMALL_RECT writeRegion 생성 후 WriteConsoleOutputA()로 charInfo 전부 출력.
- Draw에서 제공한 배열을 한 번에 콘솔 버퍼에 반영.

==================================================
요약(모듈 핵심 동작)
==================================================

- Actor/게임 로직은 Submit()로 렌더링 요청만 모음(RenderQueue).
- Draw() 시점에 가시성 검사, 클리핑, 우선순위 병합을 수행하여 프레임 버퍼를 구성.
- frame은 현재 가공 중인 논리 버퍼, ScreenBuffer 2개는 실제 콘솔 핸들 출력 버퍼.
- Present()로 출력 버퍼를 교체해 더블 버퍼링 렌더링을 수행.
